你的任务是根据设计文档开发 DIP for HR 产品中的“销售人效分析“场景 DEMO。你需要开发以下几部分内容：
1. Web 界面。严格按照设计文档的描述来进行开发，不要增加任何设计文档上没有提到的功能。
2. API 服务。你不需要实现真正的业务逻辑，只需要实现 API 接口并返回 Mock 数据。

# 文档
在开始编写代码之前，一定要仔细阅读所有有关的文档。注意，每个功能模块都不是完全独立的，你必须根据产品架构来梳理清楚各个模块之间的逻辑关系。
设计文档在 design 目录下：
* 销售人效分析.pdf：销售人效分析的需求规格说明书。

# 前端
- 使用 TypeScript、Tailwind CSS、ReactJS、Ant Design、RSBuil 作为技术栈。

# 后端
使用 Flask 作为后端服务框架。
